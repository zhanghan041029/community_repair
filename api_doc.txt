社区维修系统管理平台 - 完整接口文档
基础路径：/api（示例）
鉴权方式：除登录、注册接口外，均需在请求头携带 Authorization: Bearer <token>
统一响应格式：json
{ "code": 1, "msg": "success", "data": ... }
- code：1 成功，0 失败
- msg：提示信息
- data：返回数据，可为对象、数组或 null

1.1 用户登录
请求路径：/login
请求方式：POST
接口描述：居民、维修工、管理员通用登录，返回 token 和角色

3.4 管理员审核维修工
请求路径：/workers/review
请求方式：PUT
接口描述：管理员对待审核的维修工进行审核（通过/拒绝）
请求参数（application/json）：
参数名
类型
是否必须
备注
workerId
number
是
维修工用户ID

boolean
是
true：审核通过（状态变 active）；false：审核拒绝（状态变 rejected）
rejectReason
string
否
拒绝原因（approved=false 时可填写）
请求样例：
json
{ "workerId": 10, "approved": true }
或
json
{ "workerId": 10, "approved": false, "rejectReason": "证件不清晰" }
响应数据：仅 code/msg
3.5 删除维修工（管理员）
请求路径：/workers/{id}
请求方式：DELETE
接口描述：删除维修工账号（若其有未完成工单则禁止删除，返回失败原因）
